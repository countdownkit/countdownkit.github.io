# CLAUDE.md — countdownkit

Project instructions for Claude Code working in this repo. See `README.md` for full detail;
this file is the quick, must-know context.

## What this is

A zero-dependency static-site generator. `generate.js` reads `data/events.json` + `assets/`
and writes ~690 HTML pages into `public/`. It's a boring, SEO-driven, AdSense-monetized
"days until X" site. Live at https://countdown.elevatedprogress.com/ (GitHub Pages origin countdownkit.github.io). Audience is general/US consumers
(not developers) — keep the UI clean, fast, and light; that's deliberate (ad-friendly, low bounce).

## Deploy — just push

Deployment is automated. **`git push` to `main` is the deploy** — GitHub Actions
(`.github/workflows/deploy.yml`) runs the generator and publishes to Pages in ~1 min.

- **Never manually build and commit output.** `public/` is git-ignored build output.
- **Never hand-edit anything in `public/`** — it's regenerated every build.
- Commit as the neutral identity, NOT a personal name:
  `git -c user.name="countdownkit" -c user.email="countdownkit@users.noreply.github.com" commit …`
  (This repo is public; keep the owner's real name off it.)

## Local build / preview

```
node generate.js     # writes ./public
node server.js       # preview at http://localhost:5055
```
On Windows set config via **PowerShell** env vars, e.g.
`$env:DOMAIN='https://countdownkit.github.io'; $env:BASE=''; node generate.js`.
Do NOT set `BASE` in Git Bash — its POSIX path conversion mangles a leading-slash value.
(LF→CRLF warnings on commit are harmless.)

## Adding pages

Edit `data/events.json`. Rule types: `fixed`, `nth`, `last`, `easter`, `easter-offset`,
`offset`, `election`, `table` (explicit per-year dates). Canonical + per-year pages + internal
links are produced automatically. The every-calendar-date and N-days-from/ago families are
generated in code, independent of the data file.

**Do not add lunar/observance holidays (Hanukkah, Ramadan, Eid, Diwali…) with computed guesses.**
Only add them via a verified `table` of dates.

## Don't break these (generated by generate.js, must keep serving)

- `google78307d8185e2853c.html` — Google Search Console ownership verification. Remove it and
  the property un-verifies.
- `ads.txt` + the AdSense loader in `<head>` — publisher `ca-pub-5580575158570188`.
- `sitemap.xml`, `robots.txt`, `.nojekyll`.

## AdSense state

Account is submitted and in review. In-content ad boxes are the `.ad-slot` placeholder divs.
After approval, real `<ins class="adsbygoogle">` units (made in the AdSense dashboard) replace
those placeholders in `layout()` in `generate.js` — then push.

## Config knobs

`DOMAIN` (canonical + sitemap origin) and `BASE` (serve sub-path; `""` for the current root
domain). Production values live in the workflow. A future custom domain = add `CNAME`, keep
`BASE=""`, push.
